# Contributing to BlindBid Protocol

Thanks for your interest in contributing! Please follow this guide to ensure smooth collaboration.

## How to contribute

1. Fork the repository and create a feature branch.
2. Ensure your change is well-scoped and includes tests if applicable.
3. Follow the code style in existing files. Use TypeScript for frontend where applicable and modern JS for backend.
4. Run tests and linters:

```bash
# contracts
cd contracts && npm test

# backend
cd backend && npm run lint (if added)

# frontend
cd frontend && npm run lint
```

5. Open a pull request with a clear description and link to any relevant issues.

## PR checklist

- [ ] Tests added/updated
- [ ] Linter passes
- [ ] Documentation updated (README, examples)
- [ ] Security considerations described for substantial changes

## Code of conduct

Please follow the project's Code of Conduct (if present) and be respectful to maintainers and contributors.
